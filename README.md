# 小米笔记导出工具 v1.0.0

## 功能特性

- ✅ 批量导出小米云服务笔记
- ✅ 支持文件夹选择和全选功能
- ✅ 自动下载笔记中的图片
- ✅ 断点续传功能，支持中断后继续导出
- ✅ 反爬虫设置，可配置延迟和批量处理
- ✅ 过滤不需要的文件夹（已完成、未完成、最近删除、我的文件夹）
- ✅ 自动为当年的笔记添加年份信息
- ✅ 支持完整导出（含图片）和纯文本导出
- ✅ 导出为Markdown格式，便于Obsidian等笔记软件使用

## 安装方法

### 方法一：开发者模式安装（推荐）

1. 打开Chrome浏览器，访问 `chrome://extensions/`
2. 开启右上角的"开发者模式"
3. 点击"加载已解压的扩展程序"
4. 选择包含此插件的文件夹
5. 确认安装

### 方法二：打包安装

1. 将所有文件压缩为ZIP格式
2. 在Chrome扩展页面点击"加载已解压的扩展程序"
3. 选择ZIP文件进行安装

## 使用说明

### 基本使用流程

1. 打开小米笔记页面：https://i.mi.com/note/h5
2. 登录你的小米账号
3. 点击浏览器工具栏中的插件图标
4. 在弹出窗口中：
   - 选择要导出的文件夹
   - 设置导出选项（是否下载图片、图片文件夹名称等）
   - 配置反爬虫设置（延迟时间、批量大小等）
5. 点击"开始导出"按钮
6. 等待导出完成，查看日志了解进度

### 文件夹过滤

插件会自动过滤以下文件夹：
- 已完成
- 未完成
- 最近删除
- 我的文件夹

只导出实际的笔记文件夹，避免重复导出。

### 图片下载

图片下载使用以下技术：
- 自动识别笔记中的图片元素（`img.image-view__img`）
- 使用 `credentials: 'same-origin'` 和 `mode: 'cors'` 解决CORS问题
- 支持断点续传，避免重复下载
- 图片保存在 `images` 文件夹中

### 断点续传

插件支持断点续传功能：
- 每处理完一条笔记自动保存进度
- 如果导出中断，重新打开插件时会提示继续
- 可以选择"继续导出"或"清除记录"重新开始

### 反爬虫设置

为了防止被小米云服务检测到爬虫行为，插件提供以下设置：

- **最小延迟**：每次操作的最小延迟时间（毫秒）
- **最大延迟**：每次操作的最大延迟时间（毫秒）
- **批量大小**：处理多少条笔记后强制休息
- **批量休息**：强制休息的时间（秒）
- **笔记加载延迟**：点击笔记后等待的时间（毫秒）
- **文件夹加载延迟**：切换文件夹后等待的时间（毫秒）

建议设置：
- 最小延迟：1000ms
- 最大延迟：3000ms
- 批量大小：20条
- 批量休息：60秒

## 导出格式

### Markdown格式

导出的笔记使用Markdown格式：
- 使用二级标题（##）作为笔记标题
- 包含创建时间信息
- 图片使用Markdown语法引用
- 支持列表、标题等格式

### 文件结构

导出后的文件结构：
```
Downloads/
├── 文件夹名1.md
├── 文件夹名2.md
├── images/
│   ├── 文件夹_笔记名_img1.png
│   ├── 文件夹_笔记名_img2.png
│   └── ...
└── mi_notes_export.json（包含所有笔记的原始数据）
```

## 常见问题

### Q: 图片下载失败怎么办？

A: 插件会自动尝试以下方案：
1. 使用优化的fetch配置下载
2. 如果失败，在新标签页打开图片
3. 提示用户手动右键另存为

### Q: 如何避免被反爬虫检测？

A: 建议设置：
- 增加延迟时间
- 减小批量大小
- 增加批量休息时间
- 避免频繁操作

### Q: 导出的图片在哪里？

A: 图片默认保存在 `images` 文件夹中，可以在导出选项中修改文件夹名称。

### Q: 如何重新导出？

A: 在弹出窗口中点击"清除记录"按钮，清除之前的进度，然后重新开始导出。

### Q: 支持哪些浏览器？

A: 目前支持基于Chromium的浏览器：
- Chrome
- Edge
- Brave
- Opera
- Vivaldi

## 技术特性

- 使用Chrome Extension Manifest V3
- 支持Service Worker后台脚本
- 使用IndexedDB存储进度信息
- 使用Chrome Downloads API处理文件下载
- 使用Chrome Storage API存储配置和进度

## 隐私说明

- 插件只在本地运行，不上传任何数据到第三方服务器
- 所有笔记数据仅保存在本地Downloads文件夹
- 不收集任何用户信息或使用统计
