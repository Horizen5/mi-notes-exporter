# 小米笔记导出工具 更新日志

## [v1.9.4] - 2026-03-31

### 🔄 滑动窗口同步（Sliding Window Sync）
- **上一屏末尾锚点**：每轮提取结束时，保存当前视口中最后 3 个成功导出的 `noteId` 到 `lastAnchorIds` 数组。
- **滚动后重叠对齐**：执行滚动并等待渲染后，在新视口中寻找这 3 个 `lastAnchorIds`，跳过直到最后一个锚点为止的节点，只处理新内容。
- **连续性判定**：如果新视口完全找不到锚点 ID，提示可能存在漏单风险。

### 🔒 强化全局唯一性拦截
- 在 `processNoteItem` 函数中，执行 `notes.push` 前强制进行双重检查，确保不会重复处理同一笔记。
- 清理了主循环中重复的处理逻辑，统一使用 `processNoteItem` 函数。

### ✅ ID 稳定性优化
- 确保 `noteId` 仅由静态特征生成：标题 + 创建时间 + 内容长度 + 预览内容 + HTML 结构哈希。
- 碰撞解决器仅在同一批次内使用索引作为临时后缀，不影响最终 ID 的稳定性。

## [v1.9.1] - 2026-03-30

### ✅ 导出稳定性增强
- 文件夹切换后强制回到列表顶部，并等待列表稳定，减少“从中间开始扫”导致的漏导。
- 优化滚动容器定位：优先使用 `.note-list-items-*` 容器，滚动更准确。
- 修复列表扫描中的遗留变量问题，避免异常中断导致漏导。

### 🧠 去重逻辑增强
- 新增稳定键提取：优先读取 DOM `data-*`，其次尝试从 React 内部数据提取候选 ID，最后回退到文本组合键，降低误判重复概率。

### 🧹 架构清理
- 移除断点续传：删除进度保存/恢复与侧边栏“未完成任务”UI，仅保留暂停/继续/停止。
- 代码补充中文注释：为内容脚本与侧边栏核心流程增加模块级说明，便于阅读与维护。

### 📦 归档说明

## [v1.9.0] - 2026-03-30

### 🧩 命名与导出逻辑更新
- JSON 文件命名改为按文件夹独立导出，格式：`mi_note_文件夹名.json`，与 Markdown `文件夹名.md` 一一对应，便于按目录管理。
- 支持“停止导出”后的部分导出：自动生成带后缀的文件名（`_未全部导出`），同时在 Markdown 内部标注“部分导出”提示。
- 导出流程统一：无论任务正常完成或手动停止，都会执行文件生成与保存。

### 🔧 行为与稳定性
- 更新停止逻辑：`stopExport` 不再立即清除状态，等待主循环结束后统一执行导出与收尾。
- `generateOutput` 支持 `isPartial` 标志，文件下载函数支持动态后缀。

### 📦 归档说明
- 创建“还原点”归档：包含当前版本的目录副本与 ZIP 压缩包，并统一移动至“历史更新”目录。

## [v1.8.1] - 2026-03-30

### 🔧 小幅优化
- **导出文件命名优化**：
  - 为导出的 JSON 数据文件和 Markdown 文件夹文件添加了 ISO 格式的时间戳后缀（例如 `_2026-03-30T15-30-00`）。
  - 解决了多次导出时文件名冲突导致浏览器自动重命名的问题，使备份文件版本更清晰。

## [v1.8.0] - 2026-03-30

### 🛠️ 图片下载重构与修复
- **下载引擎升级**：彻底弃用了网页端 `<a>` 标签模拟点击下载，改为使用 **`chrome.downloads` API**。
- **子文件夹归类**：现在图片会自动保存到下载目录下的 `images/` 文件夹中（路径可自定义），解决了文件散落在根目录的问题。
- **动态格式识别**：
  - 弃用了硬编码 `.png` 的逻辑。
  - 实现了基于二进制流（MIME Type）的自动格式识别：照片存为 `.jpg`，绘图存为 `.png`。
- **Markdown 引用对齐**：确保 Markdown 文档中的图片链接与实际保存的文件名（含正确后缀）百分之百一致，解决了 Obsidian 无法预览图片的问题。
- **跨域验证优化**：采用“前端预抓取 Base64 + 后台代理保存”的组合方案，完美解决了后台下载无法携带小米登录凭证的问题。
